~3.0 Method Overloading~

As mentioned in Lab 6, polymorphism in Java can be achieved by overriding and overloading methods. This section will take a close look at method overloading.

~Task 3.3~

Method overloading is a way to achieve compile-time polymorphism or static binding.This means method calls are resolved at compile-time by the Java Compiler.

Method overloading can be done in any of the following ways:

  • Varying the number of arguments.
  • Varying the types of the arguments.
  • Changing the method return type ONLY if one of the above has also been done.
  • Unlike method overriding, overloading can be done on methods within the same class, and also superclass methods.
