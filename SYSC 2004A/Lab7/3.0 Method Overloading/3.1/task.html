~3.0 Method Overloading~

As mentioned in Lab 6, polymorphism in Java can be achieved by overriding and overloading methods. This section will take a close look at method overloading.

~Task 3.1~

Method overloading is the other way to achieve polymorphism in Java. In this task, we have the car classes from the previous section. Method overloading is useful when some function of a class should be able to use different types of data.

For example, imagine a class Bear that has a method eat(). Bear might eat Honey, or it might eat Salmon. It is unlikely that Honey and Salmon share the same inheritance tree (unless you go all the way back to Object). And so, there are three possible solutions to this problem:

  1. Make one method with a signature of - eat(Object o)
  2. Make two methods with signatures of - eat(Honey h) and eat(Salmon s)
  3. Make two methods with signatures of - eatHoney(Honey h) and eatSalmon(Salmon s)

Which solution do you prefer?
