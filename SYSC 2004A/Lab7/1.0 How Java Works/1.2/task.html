~Task 1.2~

What is the JVM (Java Virtual Machine)? Well, it is a virtual machine (sort of) that runs Java code. Using a virtual machine gives Java the advantage of being portable, meaning any OS can run Java bytecode by adhering to the JVM specifications. In short, you can take Java classes compiled on Windows, and run the compiled bytecode on Linux (for example), assuming a JVM is present.

At a high level, the specifications for a JVM are essentially: must be able to load .class file, and then execute it. These functions are performed by the JVM's class loader and execution engine respectively.
