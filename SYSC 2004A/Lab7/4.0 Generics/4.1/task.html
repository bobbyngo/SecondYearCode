~Task 4.1~

Up until now, we have often seen a collection, such as an ArrayList, parameterized with some data type upon instantiation, as so: ArrayList<String>. The type parameters are specified using angle brackets (<>), otherwise known as the diamond operator. So what is going on here?

If you take a peek at the Java code for ArrayList, LinkedList, you will see they are declared with some "type" E. E is a placeholder for a type parameter! With public class ArrayList<E> ..., we are saying that the class is generic, and is not expecting a specific type; the type is variable! We must then specify the type when we instantiate a new ArrayList: new ArrayList<String>();.

Consider the code in Main.java. Despite what you might think, this code would compile properly! This is because the compiler does not know if the type-cast to a FordBronco (line 10) is permissible or not, because we have not told it what element type to expect in the List (line 8). Parameterized types allow us to keep our generic classes, and also tell the compiler what type to expect so the correct casts can be made under the hood, and we don't get unexpected runtime errors. Otherwise known as compile-time type safety. Run Main.java as it is now and see what happens.
