~2.0 Layouts~

Layouts are the most important part of a Swing application. They determine how your application will look, and function. Complete this section to find out about how to manage the placement of components in your application.

~Task 2.1~

Our example does not look very good so far. The label is off-center and barely visible. How can we change this? Using Layouts! Checkout the available layouts here: Oracle Docs: Layout Managers.

The default layout for a JFrame is the BorderLayout. So our frame is using a BorderLayout. Looking at the documentation for BorderLayout, we can see the possible positions for components in this layout. To set the position of a component when we add it, we must specify it in the call to JFrame.add(). An example is shown in Main.java.

Important notes:

  • Notice the call to frame.setSize() is removed. We have replaced it with a call to frame.pack() farther down. In general, you should use frame.pack(). What frame.pack() does is it "squishes" all the components together and sizes them to their preferred size or greater. Where are these preferred sizes determined? Generally, through your layout manager! More on this later!
  • The call to frame.setResizeable() has also been removed so we can resize our window.
  • Notice the import of java.awt.*. This is a needed package, but you should not concern yourself too much with the details. In brief, Swing is built in part off of AWT. Specifically, the JWindow inheritance tree (JFrame, for example).
    • IMPORTANT: finally we can see why it is a good idea to be explicit in your import statements. Try importing java.awt.* and java.util.*, and then creating a new List. We get a class clash; not good! You will likely run into this issue in the Lab Problem. Be explicit in your imports: it is fine to have many import statements.
