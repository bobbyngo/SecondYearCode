~3.0 Listeners~

Visuals are great, but how do we add some functionality to an application? This is generally implemented using listeners. This section provides an introduction on how to use them.

~Task 3.1~

The first listener to be aware of is the ActionListener. It is common to attach an ActionListener to a JButton. By doing this we can define behaviour that will happen when the JButton is clicked.

In Main.java there is a method called getColourButton() which returns a JButton in the form we want. You should try to do this for any standard components that appear many times in your application. In this way you aren't writing a 3000-line main() or displayGUI() method.

You are probably wondering what the code below does. The first line is simple: we are adding a new ActionListener object. The next part is the creation of an anonymous inner class. It is called anonymous because it is not declared as a class usually is (class MyClass {}), and therefore has no name. Basically, what is happening is a subclass of ActionListener is being created, and we are giving this anonymous subclass an overriding method for the superclass' method actionPerformed(). actionPerformed() is what gets called under the hood when an event occurs (such as a button click).

Be careful! Within the context of the anonymous inner class, this no longer refers to the outer class! 

button.addActionListener(new ActionListener() {
    @Override
    public void actionPerformed(ActionEvent ae) {
        System.out.println("AHHHH");
    }
});
