~2.0 JUnit~

JUnit is a widely used unit testing framework for Java. Using JUnit is a good first step in testing your software. This section will provide an introduction to creating JUnit test classes in Java.

~Task 2.1~

JUnit tests can be used to test methods in a class. An example JUnit test class can be found in MyTests.java. You can find details here: JUnit Docs. Everything you would ever need to know about JUnit can be found in the JUnit Docs. Note: there are some differences between JUnit 4 and JUnit 5, but for our purposes it is mostly the same (the @Before annotation differs from the JUnit 5 annotations).

Tests should be located in a separate package, usually called test. The general format you should follow for JUnit test classes in this course is shown in MyTests.java:

  • If you are testing non-static methods, you will likely need to instantiate some objects. This setup is called the test fixture, and can be done using the @Before annotation.
  • @Test denotes a particular test. Each "@Test" should test a specific case or function. For example, if you want to test that a removeDuplicates() method does what it is supposed to, you would create a test called testRemoveDuplicates() to test all your test cases. You might even be more specific, creating a test method for special cases like testRemoveDuplicatesAllElementsIdentical().
  • The success or failure of a test is done using assert statements. There are many types of assert statements such as assertEquals(), assertThrows(), etc. They can all be found here: JUnit Docs.
  • Finally, if you want to perform some clean up after each test you can use @After.
