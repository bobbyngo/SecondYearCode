~1.0 Exceptions~

Knowing how to handle exceptions is exceptionally important to writing robust and stable applications. Large systems are often riddled with areas where the state, control flow, or the results of an operation are uncertain. It is important to anticipate these problem areas and handle them accordingly. This section details some fundamentals on Exceptions in Java, and how to handle them.

~Task 1.1~

At the risk of sounding redundant, everything in Java is a class, this includes Exceptions. Exception is a class from which all other exceptions are derived. It is worth mentioning that in Java, there is a difference between Exception and Error. Exception and Error are the two subclasses of Throwable. Basically, Exception is a not-so-serious problem with the code, and Error is a very-serious problem with the code. We can use and handle the former, but the latter should not be handled, and should be fixed outright.
